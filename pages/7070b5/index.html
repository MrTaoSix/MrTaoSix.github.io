# gitlab 代码提交流程

1.git branch -a 查看下远程分支 2.git branch 查看本地分支 3.git add * 将任何新的或者修改的文件添加到索引。也就是告诉git把文件添加到仓库。 4.git checkout master 切换分支（建议切换到自己的分支进行开发） 5.git commit -m "添加提交说明"将索引的当前内容与描述更改的用户和日志消息一起存储在新的提交中。也就是把文件提交到仓库。 6.git pull origin master 从远程拉取master代码，在本地解决冲突 7.git push origin dev_test:dev_test 将本地仓库分支代码提交到远端 8.在gitlab进行与master分支的合并。 9.git merge命令用于合并指定分支到当前分支上 切换到master分支，然后执行 git merge fenzhi01，表示将fenzhi01合并到master分支。 10.git branch -d name 删除分支
