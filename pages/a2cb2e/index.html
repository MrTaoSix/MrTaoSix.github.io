杭州湖畔

  • spring boot默认最大线程数，最大连接数

    • spring boot内置的web服务器是tomcat

    • 最大线程数：

      • 每一次http请求到达web服务器，web服务器都会创建一个线程来处理该请求，该参数决定了应用服务器同时可以处理多少个http请求
      • 默认200
      • 设置：server.tomcat.max-threads=1000

    • 最大连接数

      • 同一时间，tomcat能够接受的最大连接数。
      • 默认300
      • 设置：server.tomcat.max-connections=2000

    • 最大等待数

      • 当web服务的http请求达到tomcat的最大线程数时，还有新的http请求，这是tomcat会将请求放到等待队列中，最大等待数就是这个队列的最大值。如果等待队列也放满了，新的请求就会拒绝(connect refused)
      • 默认100
      • 设置：server.tomcat.accept-count=800

    • 连接数和线程数区别

      • 原来tomcat最大连接数取决于maxConnections这个值加上acceptCount这个值，在连接数达到了maxConenctions之后，tomcat仍会保持住连接，但是不处理，等待其它请求处理完毕之后才会处理这个请求。

  • mysql不走索引的一些写法

    • 例如创建a, b, c 的索引，实际上是创建了 a 和 a,b 和 a,b,c三个索引，所以如果条件顺序或不含这些字段的查询是不走索引的
    • Like 'abc%' 是走索引的，Like '%abc' 和 Like '%abc%'是不走索引的
    • 条件 id != 2 或 id <> 2 是不走索引的
    • 条件name is null 是不走索引的
    • 条件not in 不走索引

  • 下面的代码打印的结果

    • public static void main(String[] args) {
        long start = System.currentTimeMillis();

        //核心线程数 5  最大线程数也是5
        ExecutorService executorService = Executors.newFixedThreadPool(5);

        Queue<Future> queue = new LinkedList<>();

        System.out.println("执行了");

        for(int i = 0 ;i < 10;i++){
            queue.add(executorService.submit(new Runnable() {
                @Override
                public void run() {
                    try {
                        TimeUnit.SECONDS.sleep(1);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
            }));
        }

        System.out.println("任务准备完成");

        for (Future f = null;(f = queue.poll()) != null;){
            try {
                f.get();
            } catch (InterruptedException e) {
                e.printStackTrace();
            } catch (ExecutionException e) {
                e.printStackTrace();
            }
        }

        long end = System.currentTimeMillis();

        System.out.println(end - start);
    }

    • 结果是 2000，因为该线程的核心线程数大小是5，最大线程数也是5，等待队列是无限制的，提交了10个任务，同时执行10个任务时，由于最大线程数只有5，所以刚开始只能执行5个任务，另外5个任务放在等待队列中，一秒之后，在执行等待队列里的5个任务。

    • 代码缺陷

      • 如果前面的任务执行时间较长，任务就会不断堆积，会导致内存被打满。

    • 解决方案

      • 根据业务场景或者服务器内存大小 设置一个合理的等待队列大小，并设置如果队列打满后的 处理策略，如：消息丢弃，报警等

  • 工作中遇到最大的问题，如何解决的

    • 场景
      • 和微信对接支付，有一个功能是充值，例如充值100元，微信支付后，页面显示已成功，微信回调通知成功，回调接口报错，导致账户100元没有加上，于是导致用户充值了100元，页面账户未到账的情况，排查问题后发现是回调里有个发送短信的功能，第三方报错了。
      • 解决：
        • 发送短信改成异步执行，结果不影响回调结果
        • 回调接口加上 主动查询微信订单状态的逻辑
