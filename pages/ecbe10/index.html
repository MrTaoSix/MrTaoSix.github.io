# Quartz学习笔记

版本：

# 1.1 简介

Quartz是一个任务调度框架，用于定时执行任务

使用场景：定时自动还款，隔一个小时备份一份数

# 1.2 作业持久性

Quartz的设计包括一个作业存储接口，有多种时间

  • 数据库存储：JDBCJobStore 存储在JDBC关系型数据库中
  • RAMJobStore 存储在缓存中，缺点是重启服务器会丢失任务

# 二 Quartz使用

话不多说，先上手。

# 2.1导入依赖

        <!-- https://mvnrepository.com/artifact/org.quartz-scheduler/quartz -->
        <dependency>
            <groupId>org.quartz-scheduler</groupId>
            <artifactId>quartz</artifactId>
            <version>2.3.0</version>
        </dependency>

# 2.2 定义Job

public class PrintWordsJob implements Job {
    @Override
    public void execute(JobExecutionContext jobExecutionContext) throws JobExecutionException {

        JobDetail jobDetail = jobExecutionContext.getJobDetail();
        String name = jobDetail.getKey().getName();
        String group = jobDetail.getKey().getGroup();
        String job = (String) jobDetail.getJobDataMap().get("data");
        String printTime = new SimpleDateFormat("yy-MM-dd HH-mm-ss").format(new Date());
        System.out.println("PrintWordsJob start at:" + printTime + ", prints: Hello Job-" + new Random().nextInt(100));

    }
}

定义一个Job类来执行具体的工作，实现Job

# 2.3 API 测试

public class demo {

    public static void main(String[] args) throws SchedulerException, InterruptedException, ParseException {


        //1 创建调度器
        SchedulerFactory schedulerFactory = new StdSchedulerFactory();
        Scheduler scheduler = schedulerFactory.getScheduler();

        //2. 创建JobDetail，并绑定任务
        JobDetail jobDetail = JobBuilder.newJob(PrintWordsJob.class)
                .withIdentity("Job1","group1").build();

        //3 构建Trigger实例，每隔一秒执行一次
        Trigger trigger = TriggerBuilder.newTrigger().withIdentity("trigger1","triggerGroup1")
                .startNow()
                .withSchedule(SimpleScheduleBuilder.simpleSchedule()
                        .withIntervalInSeconds(1)//间隔每秒
                        .repeatForever())//一直执行 直到结束时间
                .endAt(new SimpleDateFormat("yy-MM-dd HH-mm-ss").parse("2021-05-24 18-05-00"))//结束时间
                .build();
        //4 执行
        scheduler.scheduleJob(jobDetail,trigger);
        System.out.println("--------scheduler start ! ------------");
        scheduler.start();

    }

}

# 2.4 配置

spring:
  quartz:
    properties:
      org:
        quarta:
          scheduler:
            instanceName: DefaultQuartzScheduler # 调度器名称
          threadPool:
            class: org.quartz.simpleThreadPool  # 线程池
            threadCount: 10 # 线程池大小
            threadPriority: 5 # 优先级
          jobStore: # 存储类型 
            class: org.quartz.simple.RAMJobStore

# 2.5 核心类说明

  • Scheduler: 调度器，所有的调度都是由它控制，Quartz的大脑
    • JobStore : 存储运行时的信息，包括触发器，JobDetail等
    • ThreadPool:线程池，所有任务由线程池去执行

# 三 Trigger 触发器

# 3.1 SimpleTriger 毫秒间隔

简单触发器，指定从某一个时间开始，以一定的时间间隔（毫秒）执行的任务。

类似场景：9:00开始每隔一个小时，执行一次

属性如下：

  • repeatInterval:重复间隔

  •         SimpleScheduleBuilder.simpleSchedule()
                .withIntervalInSeconds(3)//间隔三秒执行一次
                .repeatForever()//一直执行下去
                .build();

  • repeatCount : 重复次数，实际执行的是参数+1，因为在启动的时候会执行一次。

  •         SimpleScheduleBuilder.simpleSchedule()
                .withIntervalInSeconds(3)//间隔三秒执行一次
                .withRepeatCount(3)//执行三次
                .build();

# 3.2 CalendarIntervalTrigger 分钟～年间隔

类似于SimpleTrigger，指定从某一个时间开始，以一定时间的间隔执行任务，但是不同的是间隔单位不是毫秒，它的间隔单位有秒，分钟，小时，天，月，年，星期，因为simple没办法用月来间隔（每个月的间隔不一样）

代码示例：

        CalendarIntervalScheduleBuilder.calendarIntervalSchedule()
                .withIntervalInDays(2)//每隔两天执行一次
                .build();


        CalendarIntervalScheduleBuilder.calendarIntervalSchedule()
                .withIntervalInWeeks(1)//每隔一个星期执行一次
                .build();

# 3.3 DailyTimeintervalTrigger 每天的某个时间段内

指定每天的某个时间段内，以一定的时间间隔开始执行任务，可以指定星期。

使用场景：指定每天9:00 至 18:00 ，每隔10秒执行一次，并且只要周一到周五执行

像不像每天监督你工作的老板😊

属性如下：

  • startTimeOfDay 每天开始时间
  • endTimeOfDay 每天的结束时间
  • dayOfWeek 需要执行的星期
  • Interval 执行间隔
  • intervalUnit 执行间隔的单位（Y,d,H,M,s...）
  • repeatCount: 重复的次数
        DailyTimeIntervalScheduleBuilder.dailyTimeIntervalSchedule()
                .startingDailyAt(TimeOfDay.hourAndMinuteOfDay(9,0))// 九点开始
                .endingDailyAt(TimeOfDay.hourAndMinuteOfDay(18,0))//18点结束
                .onDaysOfTheWeek(DailyTimeIntervalScheduleBuilder.MONDAY_THROUGH_FRIDAY) //周一至周五执行
                .withIntervalInHours(1)//间隔一个小时
                .withRepeatCount(100)//执行一百次
                .build();

# 3.4 CronTrigger (重点)

感觉上面的好麻烦，要记那么多东西，那么简单的来了

这个触发器就牛逼了，他是个更负责的任务，支持linux Cron的语法，基本覆盖其他三种的功能，但不是全部。

Cron表达式，

       CronScheduleBuilder.cronSchedule("0/1 * * * * ? ")
                .build();

这个表达式不多记录，懂的都懂 不懂百度cron
