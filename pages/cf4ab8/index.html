视图&&存储过程

# 1. 视图

视图是一个虚拟表，是s q l的查询结果。同真实的表一样，包含带有名称的列和数据。视图本身不包含数据，返回都都是从其他表中检索出来的数据

视图的优点：

  • 数据独立：一旦视图结构确定，可以屏蔽表结构对数据的影响
  • 安全：用户只能看到视图中的数据
  • 简单：不需要关心视图中的数据是如何获取的

视图的限制：

  • 不能建立索引
  • 在from字句中不能使用子查询

# 1.1 创建视图

mysql> create view v1 as select * from iot_risk_customer limit 10;
Query OK, 0 rows affected (0.07 sec)

如果在视图表中不定义字段名，那么默认为基表的字段名,如果需要定义字段名。字段数量需要与基表一样。

  • 查看所有表的状态

    mysql> show table status\G;
# Comment: VIEW
# 注释字段为VIEW为视图表

  • 显示创建的命令

    mysql> show create view v1\G;
mysql> show table status where comment="VIEW"\G;

    视图表和基表的数据是会互相影响的，任何一方修改都会影响另外一方

  • 覆盖创建视图

    create or replace view v3 ： 当创建的视图名已存在，该选项可以覆盖）

# 2. 存储过程

存储过程，相当于s q l 语句组成的脚本，指的是数据库中保存的一系列s q l命令的集合，可以在存储过程中使用变量，条件，流程控制等

好处就是：

提高性能，防止对表直接访问，避免重复编写s q li
