  • spring cloud哪些组件

    • Netflix Eureka 服务发现
      • 注册表，用来注册所有服务信息

    • Netfilx Ribbon 客户端负载均衡，策略：轮训，权重，ip
    • Netflix Hystrix 断路器
    • Netflix Zuul 服务网关
    • Spring Cloud Config 分布式配置

  • object哪些常用方法

    • equals
      • Object是所有类的父级类，对象和对象的值比较都是调用equals方法比较，如果某类没有重写equals方法，用equals方法比较和==效果一样，反之按照重写equals方法里的内容进行比较值。
    • hashCode
      • 集合类如 hashmap中，判断某个位置的key值是否已经存在是先根据对象的hashcode码比较，在根据对象的equals方法比较: o1.hashcode = o2.hashcode && o1.equals(o2)
      • 为什么重写equals方法的同时也要重写hashcode方法
        • 如果没有重写hashcode方法会导致equals相等的对象，hashcode不相等，两个相同的对象插入hashmap时去重不了
        • o1.hashcode = o2.hashcode && o1.equals(o2) 如果hashcode不同，则不用在比较equals方法了，提高性能
    • tostring
      • 打印对象的属性
    • wait
      • 让当前线程从运行状态 -> 等待队列
    • notify
      • 从等待队列 -> 锁池
    • notifyAll

  • 并发类有哪些

    • copyOnWriteArrayList
    • copyOnWriteArraySet
    • ConcurrentHashMap
    • ConcurrentSkipListMap
    • ConcurrentSkipSetMap
    • ReentrantLock

  • feion调用原理

    • 项目启动时，被@feionClient注解修饰的接口会生成一个代理类
    • 调用接口的方法实际上就是调用代理类的方法，代理类会根据接口上的@requestMapping注解来动态构造出请求服务的地址
    • 最后根据这个地址完成请求，响应

  • 本地缓存问题

    • 固定不变的数据，用currentHashMap存储
    • currentHashMap底层实现了同步锁,ReentrantLock实现

  • spring boot tomcat 最多允许多少个线程

    • maxConnections 1000
    • maxThreads 200

  • eureka为什么要开启两个线程检测心跳

    • 每个eureka client启动都会创建一个HeartBeatThread的心跳线程，保证默认30秒向eureka server发送一个信息的请求，告诉server当前client还活着。
    • eureka server爱启动的时候每60秒遍历一次注册表中的信息，查看注册表中的信息是否有过期的，如果过期会加入到一个列表单独处理。

  • hashCode和equals方法的作用

    • 插入无序集合时会根据对象的hashcode方法来判断对象加入的位置，和该位置上的对象的hashcode值做比较，若hashcode相同，在判断Equals方法是否返回true，若返回true则认为该对象重复了，则不会插入集合
    • 判断两个对象的值是否相同，用equals方法
    • 两个对象相等，那么hashcode肯定相同
    • hashcode相同，对象也不一定相等

  • boot为什么会有两个配置文件，bootstrap.yml Application.yml

    • 加载顺序不一样
      • bootstrap先加载，application后加载
    • bootstrap用于应用上下文的引导阶段，配置系统级别的参数,application是由springApplicationContext加载的，配置的是应用级别的参数。
    • 属性覆盖问题
      • 相同的参数application不会覆盖bootstrap里的
      • application-dev.yml能覆盖
